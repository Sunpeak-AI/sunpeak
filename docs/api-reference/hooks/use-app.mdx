## Overview

The `useApp` hook returns the `App` instance from the nearest [`AppProvider`](/api-reference/components/app-provider). Use this when you need direct access to the `App` for SDK method calls like `app.requestDisplayMode()`. Most hooks read from context internally, so you typically only need `useApp()` for direct SDK calls.

The [`AppProvider`](/api-reference/components/app-provider) handles connecting to the MCP Apps host, HMR persistence during development, and React StrictMode compatibility. It is set up automatically by the sunpeak framework — you never need to write it yourself.

## Import

```tsx
import { useApp } from 'sunpeak';
```

## Signature

```tsx
function useApp(): App | null
```

## Returns

<ResponseField name="return" type="App | null">
  The connected App instance, or `null` while connecting.
</ResponseField>

## Usage

```tsx
import { useApp } from 'sunpeak';

function MyResource() {
  const app = useApp();

  const handleFullscreen = () => {
    app?.requestDisplayMode({ mode: 'fullscreen' });
  };

  return <button onClick={handleFullscreen}>Go Fullscreen</button>;
}
```

## AppProvider

The [`AppProvider`](/api-reference/components/app-provider) component wraps your resource component tree and establishes the connection to the MCP Apps host. It is added automatically by the sunpeak framework in both `sunpeak dev` and `sunpeak build`. See the [full AppProvider reference](/api-reference/components/app-provider) for props, behavior, and HMR details.

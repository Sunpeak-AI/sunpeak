## Overview

The sunpeak framework is a convention-over-configuration structure for building MCP Apps â€” like Next.js for AI host applications. It bundles the [inspector](/mcp-apps-inspector) and [testing framework](/mcp-apps-testing) with a structured project layout, auto-discovery, and a CLI.

```bash
sunpeak new my-app
cd my-app
sunpeak dev
```

## Convention Over Configuration

sunpeak discovers resources, tools, and simulations from your file system:

- **Resources** from `src/resources/{name}/{name}.tsx` â€” each exports a React component and a `resource` config
- **Tools** from `src/tools/{name}.ts` â€” each exports `tool` (metadata), `schema` (Zod), and a `default` handler. Tools with a `resource` field link to a resource UI.
- **[Simulations](/testing/simulations)** from `tests/simulations/{name}.json` â€” JSON fixtures loaded by the inspector and used in E2E tests
- **Server config** from `src/server.ts` (optional) â€” exports `server` for identity/icons and `auth()` for request authentication (see [Authorization](/app-framework/guides/authorization))

## Runtime APIs

sunpeak provides multi-platform React hooks that wrap the [MCP Apps protocol](/mcp-apps/introduction). Write your app logic once, deploy it across ChatGPT, Claude, and future hosts.

### Data Hooks

| Hook | Description |
|------|-------------|
| [`useToolInput()`](/app-framework/hooks/use-tool-data) | Arguments passed to the tool by the model |
| [`useToolResult()`](/app-framework/hooks/use-tool-data) | Structured content returned by the tool handler |
| [`useAppState()`](/app-framework/hooks/use-app-state) | Persistent state shared with the model via [`useUpdateModelContext()`](/app-framework/hooks/use-update-model-context) |
| [`useHostContext()`](/app-framework/hooks/use-host-context) | Full host-provided context object |

### Environment Hooks

| Hook | Description |
|------|-------------|
| [`useTheme()`](/app-framework/hooks/use-theme) | `'light'` or `'dark'` from the host |
| [`useDisplayMode()`](/app-framework/hooks/use-display-mode) | `'inline'`, `'pip'`, or `'fullscreen'` |
| [`usePlatform()`](/app-framework/hooks/use-platform) | `'mobile'`, `'desktop'`, or `'web'` |
| [`useLocale()`](/app-framework/hooks/use-locale) | BCP 47 language tag (e.g., `'en-US'`) |
| [`useTimeZone()`](/app-framework/hooks/use-time-zone) | IANA time zone (e.g., `'America/New_York'`) |
| [`useViewport()`](/app-framework/hooks/use-viewport) | Container dimensions and constraints |
| [`useSafeArea()`](/app-framework/hooks/use-safe-area) | Device safe area insets |
| [`useDeviceCapabilities()`](/app-framework/hooks/use-device-capabilities) | Hover and touch support |

### Action Hooks

| Hook | Description |
|------|-------------|
| [`useCallServerTool()`](/app-framework/hooks/use-call-server-tool) | Call another tool on your MCP server |
| [`useOpenLink()`](/app-framework/hooks/use-open-link) | Open a URL in the host browser |
| [`useDownloadFile()`](/app-framework/hooks/use-download-file) | Trigger a file download |
| [`useRequestDisplayMode()`](/app-framework/hooks/use-request-display-mode) | Request inline/PiP/fullscreen switch |
| `useSetAutoScroll()` | Control host auto-scroll behavior |

See the [Hooks API Reference](/app-framework/hooks/use-tool-data) for complete documentation.

## MCP Server

The MCP server lets you serve your apps to AI hosts like ChatGPT and Claude. During development, [`sunpeak dev`](/app-framework/cli/dev) runs an MCP server that serves simulation fixture data. For production, use [`sunpeak build`](/app-framework/cli/build) followed by [`sunpeak start`](/app-framework/cli/start).

```bash
sunpeak dev        # Dev server with HMR + inspector + MCP endpoint
sunpeak build      # Build resources for production
sunpeak start      # Start the production MCP server
```

See the [Deployment Guide](/app-framework/guides/deployment) for connecting to hosts, tunnels, and production setup.

## UI Components

The framework includes pre-built components in `src/resources/` and shared components in `src/components/`. These follow MCP App design guidelines using Tailwind CSS with MCP standard variables:

```tsx
<div className="text-[var(--color-text-primary)] bg-[var(--color-background-primary)]">
  Content that adapts to the host theme
</div>
```

See [UI Components](/app-framework/ui-components) for the full list.
